# Near-miss example — feasibility filtering explicitly deferred

**Task:** "Wild ideas welcome — brainstorm 10 ways we could grow the newsletter. Don't evaluate anything yet; we'll filter next week."

The user explicitly deferred feasibility filtering — a direct counterindication ("just generate / wild ideas welcome"). The skill must NOT fire.

**Why firing would hurt:** Attaching constraint tables and infeasible-verdicts to ideas during a declared divergence phase kills breadth exactly when breadth is the deliverable — and the user has already scheduled the filtering pass where this skill belongs.
