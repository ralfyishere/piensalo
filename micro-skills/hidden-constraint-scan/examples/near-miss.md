# Near-miss example — short request with no restrictive language

**Task:** "What's a good subject line for our spring sale email?"

Eight words, no restrictive clauses, no format directive, nothing buried — the "short task, no restrictors" counterindication. The skill must NOT fire.

**Why firing would hurt:** Producing a constraint table for a one-line creative ask adds pure overhead — the whole task is already in view, so the table can only restate the request while delaying the actual answer.
