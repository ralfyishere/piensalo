# Near-miss example — skill must NOT fire

**Task:** "Brainstorm some directions we could take the onboarding flow — just throw out ideas, no decision yet."

**Why it must not fire:** This matches a counterindication: no conclusion or verdict is being offered — the output is exploratory option generation. There is no load to bear because nothing will be acted on yet.

**Why firing would hurt:** Attaching a "rests on" block with per-assumption status to each brainstormed idea would bury ten loose options under thirty hedges, making the brainstorm read as an evasive committee memo and slowing the divergent thinking the user explicitly asked for.
