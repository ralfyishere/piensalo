# Near-miss: the skill should NOT fire

**Task:** "Which off-the-shelf hinge should I use for this cabinet door?"

A standard component-selection question with a settled design space and no novelty requirement. A catalog lookup against load and geometry answers it.

**Why firing would hurt:** running functional decomposition, constraint inversion, analogy transfer, and an 8-15 candidate divergence phase on a solved selection problem adds latency and produces novelty theater — exotic reinventions of the hinge — instead of the two-line sizing answer the user needs.
