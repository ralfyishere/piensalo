# Near-miss: when fable-verify should NOT fire

**Task:** "Does `len([1, 2, 3])` return 3 in Python?"

The claim is trivially and deterministically checkable in one step; there is no candidate artifact whose hidden failure modes need adversarial probing.

Why firing would hurt: a full disqualifier sweep and per-criterion verdict table for a one-line fact doubles latency and produces ceremony instead of the single check that settles it.
