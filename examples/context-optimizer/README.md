# PIENSALO Context Optimizer — demo

Task-specific context optimization with verified quality: build the
smallest context packet that preserves what THIS task needs (mandatory
constraints, exact identifiers, relevant evidence), send it to a model,
verify the response deterministically, and expand or fall back when
verification fails.

## Run it

```sh
./demo.sh
```

Step 1 (`piensalo context optimize`) is offline, deterministic, and
model-independent — the anti-drift test regenerates it byte-for-byte.
Step 2 (`piensalo context evaluate`) calls a real model through the
explicitly configured `claude-cli` adapter and compares full context vs
optimized context with the same deterministic grader on both arms.

## What the committed run shows (one real recorded run)

Generated by this script with `claude-haiku-4-5-20251001` (single sample;
the adapter exposes no temperature/seed control — outcomes vary between
runs and the evaluation records exactly what happened):

| | full context | optimized context |
|---|---|---|
| prompt tokens (est) | 2,007 | 649 |
| verification | failed all 7 requirements | passed all 7 |
| outcome | asked a clarifying question instead of answering | complete correct answer |

- Gross context reduction: **67.7%** (1,807 → 496 estimated tokens even
  before the contract envelope); runtime net input savings est **65.8%**.
- Verdict: **IMPROVED** — the optimized packet passed requirements the
  full-context response failed, with zero regressions. In this run the
  distractor-heavy full context confused the model into refusing;
  removing irrelevant context made the answer *better*, not just cheaper.
- Expansions: 0 · Fallback: none needed.

This is one sample on one task — not proof of universal improvement. The
pre-registered eight-task evidence suite lives in
`evals/context-optimizer/` (results committed under `results/`): median
gross reduction 80.2%, median runtime net input savings 76.9%, 7/7
optimizable tasks MAINTAINED, zero critical regressions, and the
uncompressible task refused optimization and fell back safely.

## Honest limits

- Quality verdicts cover the deterministic graders (output contract +
  exact-content oracle). Broader behavioral equivalence remains
  UNMEASURED.
- Token counts are estimates (chars/4) plus adapter-billed totals; the
  claude CLI carries constant harness overhead in both arms, reported
  separately in the ledgers.
- When verification cannot accept an optimized response within the
  bounded expansion limit, the outcome is a SAFE FALLBACK to full
  context — never a silently degraded answer.
